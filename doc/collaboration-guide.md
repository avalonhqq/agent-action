# 大模型专项学习协作规则

## 学习问答归档

每个检查点提出的思考题，在实现或评审后由 Codex 给出参考答案，并写入当周唯一学习记录。答案
需要说明设计理由、边界和取舍；后续结论变化时追加修订记录，不直接删除历史结论。

## 课程原则

学习者只重点投入大模型、RAG、检索、Agent、安全和评估。支撑这些能力所需的普通工程代码由 Codex 自动完成并校验。

## Codex 负责

- 自动实现数据库、迁移、CRUD、鉴权接线、普通 API、页面和部署脚手架。
- 为自动实现补充单元测试、集成测试、配置、错误处理和文档。
- 按风险运行 Ruff、strict mypy、pytest 和专项检查；测试结果不阻塞学习进度。
- 讲解大模型核心原理、方案选择、指标和失败样本。
- 提供 Prompt、Schema、Policy、Graph 和评估目标。
- 保存每次实现过程、问题、决策、指标和复盘记录。

## 学习者负责

- 理解并回答大模型核心问题。
- 参与 Intent Schema、Prompt、Chunk、RetrievalPolicy、Agent State 等 AI 设计。
- 阅读评估报告，分析错误来自意图、召回、重排、生成、工具还是安全层。
- 对 AI 核心代码做针对性实验；无需完成重复 CRUD 和部署代码。
- 对业务目标、风险阈值和效果取舍提出意见。

## 两类任务标记

- `AI 核心`：需要重点讲解、讨论和实验。
- `自动底座`：Codex 直接实现、测试并留档，学习者只需了解接口和为何存在。

## 新的模块流程

```text
确认 AI 目标
→ Codex 自动完成基础依赖
→ 记录工程检查结果（非学习阻塞项）
→ 讲解 AI 原理与 Schema
→ 实现/实验 Prompt、Policy 或 Graph
→ 运行固定评估集
→ 分析失败样本
→ 版本迭代与留档
```

## 沟通方式

每个模块开始时说明：

- AI 学习目标。
- 自动完成哪些基础设施。
- 本模块实验变量和固定不变量。
- 评估数据、指标、最低验收线和主要风险。

每个模块完成时说明：

- 新增能力。
- 指标变化和失败样本。
- Prompt/Schema/Policy/Graph 版本。
- 自动底座和质量门禁结果。
- 已知限制和下一实验。

## 质量纪律

- 自动完成不等于跳过测试或直接标记完成。
- AI 效果不能只靠演示案例，必须有固定评估集。
- 不为了提高单一指标破坏安全、权限或无证据拒答。
- 不保存或要求模型输出私有思维链，只保存证据、结构化决策和原因码。
- 未通过工程门禁或专项评估时，模块不得标记完成。
