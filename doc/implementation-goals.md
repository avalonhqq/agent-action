# BiliSupport AI 最终项目目标

## 1. 产品定位

构建一个全 Python 实现、可演示、可评估、可部署的仿哔哩哔哩企业多 Agent 智能客服网站。

系统服务六类核心场景：

| 领域 | 示例 | 处理方式 |
|---|---|---|
| 账号安全 | 登录、冻结、换绑 | 知识 + Mock 身份校验 + 人工技能组 |
| 会员订单 | 权益、续费、退款 | 知识 + 受控订单工具 |
| 创作者 | 投稿、审核、版权 | 知识 + Mock 稿件工具 |
| 社区规范 | 举报、处罚、申诉 | 规则 RAG + Mock 处罚工具 |
| 技术支持 | 播放、弹幕、客户端 | 故障知识库 + 排障流程 |
| 普通咨询 | 活动、功能、帮助中心 | 标准问 + 混合 RAG |

## 2. 最终用户能力

- 网站端多轮、流式客服对话。
- 查看历史会话和继续咨询。
- 获得带标题、章节、页码和版本的知识答案。
- 中置信度时从候选问题中选择。
- 查询本人仿真会员、订单、退款、稿件和处罚信息。
- 高风险写操作必须二次确认。
- 低置信度或高风险问题生成结构化人工交接包。

## 3. 最终运营能力

- 上传并管理 PDF、DOCX、Markdown、TXT、CSV 等知识文件。
- 查看解析、分块、索引任务状态和失败原因。
- 管理标准问、相似问、审核答案、版本和有效期。
- 调试 BM25、向量、融合及重排结果。
- 查看未命中问题、负反馈和人工转接原因。
- 一键运行意图、检索、回答、工具和安全评估。

## 4. AI 能力目标

- OpenAI-compatible LLM Provider、Prompt 模板、结构化输出和 SSE。
- Small-to-Big Parent/Child Chunk、标准问、中文 BM25、向量召回、RRF、去重和批量 Reranker。
- 意图驱动、版本化 RetrievalPolicy，以及多实体证据覆盖保障。
- Intent、Knowledge、领域 Agent、Verification 与 Supervisor。
- LangGraph 确定性路由、Checkpoint、中断恢复和循环保护。
- 高分直答、中分候选、低分澄清/人工的三段式策略。
- 证据约束、引用校验、Prompt Injection 防护和幻觉降级。

## 5. Python 工程目标

- FastAPI、Pydantic v2、SQLAlchemy 2.0、Alembic。
- `asyncio`、异步 HTTP、SSE 和任务取消。
- PostgreSQL；本地可使用兼容开发配置。
- FAISS 起步，向量层可替换 Qdrant。
- NiceGUI 构建聊天、知识管理和评估页面。
- pytest、Ruff、mypy、pre-commit。
- structlog、OpenTelemetry、指标与审计。
- Docker、Docker Compose 和运行手册。

## 6. 可靠性边界

系统必须区分：

```text
静态知识问题 → RAG 证据
实时业务查询 → 受控工具
高风险业务操作 → 权限 + 归属 + 二次确认
信息不足/低置信度 → 澄清、拒答或人工技能组
```

禁止：

- Agent 自由执行任意 SQL。
- 模型猜测工具执行结果。
- 无证据生成政策结论。
- 未授权 Agent 调用业务工具。
- Mock 结果伪装为真实生产数据。

## 7. 人工服务边界

只实现技能组接入接口：判断是否转人工、选择技能组、生成交接摘要、调用 Mock 服务、保存工单编号。

不实现完整坐席调度、饱和度算法和 Redis 排队系统。

## 8. 最终成功标准

- 网站可完成知识问答、业务查询和 Mock 人工转接。
- 所有知识结论有可访问引用。
- 越权和非本人数据查询被拦截。
- 固定评估集可量化检索与回答质量。
- 检索调试结果可展示 Child 命中、Parent 上下文、通道分数、策略版本和降级状态。
- 单次请求可追踪 Agent、Retriever、LLM 和 Tool。
- 模型或外部服务不可用时有明确降级。
- Docker Compose 一条命令启动。
- README 足以让第三方独立运行和演示。
